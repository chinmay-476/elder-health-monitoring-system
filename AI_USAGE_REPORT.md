# AI Usage Report

## Project

Elder Health Monitoring System

## How AI Was Used

AI was used as a chat-based assistant during the project for guidance, debugging support, and review of architecture decisions. It was mainly helpful for:

- brainstorming folder structure
- checking JWT authentication flow
- reviewing role-based access patterns
- verifying alert rule logic
- improving documentation and deployment instructions

## How I Worked With The Suggestions

I reviewed the suggestions before using them, adjusted naming and workflow to fit the assignment, and kept the final structure simple enough to explain clearly. I also revised parts of the code and documentation to match the required roles and API flow.

## What I Personally Understood And Verified

- how JWT authentication works in this project
- how passwords are hashed before saving
- how role middleware restricts write access
- how the health alert conditions are generated in the backend
- how the frontend reads the token and calls protected APIs
- how the Render and Vercel deployment configuration works

## AI Was Not Used For

- automatically submitting the assignment
- hiding business logic from my understanding
- replacing my ability to explain the code during review

## Final Note

I used AI as a learning and productivity aid, but I made sure I understood the architecture, the role-based workflow, and the purpose of each major file before finalizing the project.
